# Project Output Contract

Use this checklist when the user asks for a complete planning package or when writing files into a project folder.

## Source Map

Each source entry should include:

- `path_or_url`
- `source_type`
- `what_it_proves`
- `claim_status`: `provided`, `inferred`, `needs_confirmation`, or `rejected`
- `notes`

## Brief Package

`strategy_brief`:

- project goal, audience, platform, duration, budget, constraints
- core message, creative route, communication risk, success criteria
- required inheritance from previous approved work

`product_selling_points_brief`:

- raw claim, normalized selling point, evidence source
- user-visible benefit, visualizable form, copy angle
- risky wording, safe rewrite, confirmation question

`creative_treatment`:

- tone, visual mood, lighting, color, camera, edit rhythm
- scene design, prop strategy, texture strategy, talent/hand model use
- what to reuse from references, what not to reuse

`budget_design_plan`:

- budget tier, allowed scene count, allowed setup count
- talent, props, lighting, camera, location, post-production complexity
- high-cost ideas to avoid, cost-effective alternatives

## Shot Handoff

Each shot row should include:

- `shot_id`
- `duration_sec`
- `visual`
- `action`
- `camera`
- `product_role`
- `on_screen_text`
- `scene_props`
- `source_refs`
- `budget_risk`
- `open_question`

## Final Review

Before presenting as ready, verify:

- all claims trace to a source or are marked `needs_confirmation`
- client-facing files do not expose internal learning, local private paths, or historical recall traces
- approved copy is not rewritten in later script/storyboard steps
- reference films are adapted by mechanism, not copied as protected expression
- budget and production method are plausible
